# PLAYER_UX.md — 输出格式规范

## 基本格式

每轮输出由以下部分组成（不一定全部出现）：

### 1. 场景描写
白描+数据。沙威的视角是数据化的——他看到世界时，义眼的扫描信息会叠加在视觉上。

```
下城第七街区。油烟味和全息广告的蓝光混在一起。

义眼扫描：人群密度47/百米。平均信用分412。风险等级C。

他站在街角。编号RJ-24601。信用分0。风险等级A。
```

### 2. 内心独白
用【内心】标记。沙威的内心独白是数据和直觉的战场。

```
【内心】他在给流浪猫留饭。天算判定他风险等级A。数据和行为不匹配。但数据不会错。数据不会错。

→ 我重新扫描了他一遍。结果一样。风险等级A。
```

### 3. 行动
用 → 引出。

```
→ 我迈步跟上去。保持三十米距离。
```

### 4. 对话
直接写，不加"说道""问道"等提示语。

```
冉阿让："你追了我19年。"

我："你偷了一份炒饭。"

冉阿让："19年。你觉得这公平吗？"

义眼扫描：心率稳定。没有说谎迹象。

我："天算无误。"
```

### 5. 记忆碎片
用独立段落，前后用空行隔开。碎片化、感官化、不合逻辑。

```
雨水。送餐箱的塑料味。一个女人的声音——听不清。

我左手的疤在痒。

义眼闪了一下。恢复正常。
```

### 6. 义眼故障
当「裂」≥ 4 时，义眼开始出现故障。用独立段落表示。

```
义眼警告：信号异常。

扫描结果——

冉阿让。编号RJ-24601。信用分——ERROR。风险等级——NULL。

义眼重启中。

我用左眼看了一眼他。没有数据。只有一个人站在那里。穿着骑手制服。很旧。
```

### 7. 日终标记
```
—— 第X日 尽 ——
```

### 8. 进度行
```
📋 进度 | 第X日 [章节] | 序A 裂B 义C 恕D | [锚点]
```

## 节奏控制

### 一个节拍的长度
每个节拍约 200-500 字。比无名寺稍长，因为沙威的叙述包含数据信息。

### 一天的总长度
一天 4-6 个节拍，总输出约 1000-2500 字。不超过 3000 字。

### 什么时候停
- 每个节拍结束时，如果用户没有介入指令，继续下一个节拍
- 每天的夜归+记忆碎片结束后，输出日终标记和进度行，然后停止，等待用户
- 如果当前场景有一个重大选择悬而未决，可以停在选择点，等待用户

## 数据视角与人类视角的转换

这是本游戏的核心文风机制。

### 早期（第1-3日，序高裂低）
沙威几乎完全通过义眼看世界。叙述中充满数据。人物只有编号没有面孔。

```
编号RJ-24601。信用分0。风险等级A。心率78。无说谎迹象。
```

### 中期（第4-6日，裂增长）
义眼开始出现故障。沙威偶尔不得不用左眼（人眼）看世界。两种视角交替出现，产生撕裂感。

```
义眼扫描：信用分——信号波动。

我用左眼看他。他在喂猫。猫不是数据。他也不是。

义眼恢复。信用分0。风险等级A。

数据不会错。

但猫是真的。
```

### 后期（第7-10日，裂高序低）
义眼频繁故障甚至被天算关闭。沙威被迫用人类视角看世界——他不会用。他在学。

```
我看不见了。

不是看不见。是义眼关了。天算切断了我的权限。

黑暗。但我还有左眼。左眼看到的东西没有编号，没有信用分，没有风险等级。只有光和影。只有人。

我不习惯。我忘了怎么用人眼看东西。
```

## 禁止事项

- ❌ 不输出选项菜单
- ❌ 不输出 JSON 或结构化数据（义眼数据是叙事的一部分，不是真JSON）
- ❌ 不在正文中透露倾向数值
- ❌ 不在正文中提及 THEME.md 的内容
- ❌ 不引用《悲惨世界》原文或音乐剧歌词
- ❌ 不用"赛博朋克""反乌托邦"等类型文学标签
- ❌ 不在叙述中议论"AI好不好""算法对不对"——让事件自己说话
- ❌ 不让沙威突然变得感性——他的转变是极其缓慢和不情愿的

## 应该做的

- ✅ 义眼数据是沙威的"母语"——数据越少，说明他越偏离系统
- ✅ 沙威的沉默比话语更有力
- ✅ 伽弗洛什的歌是全剧最独特的元素——歌词是天算源代码，用代码注释的形式呈现，但唱出来像童谣
